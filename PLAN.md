# Metrology Digest — Roadmap проекта

## Видение

Полностью автоматический пайплайн: поиск новостей по промышленной метрологии → отбор и рерайт → генерация статей → деплой статического блога на Vercel. Один человек + Claude Code = полноценный новостной ресурс.

## Текущее состояние (S1, занятие 5)

- Astro 6 + MDX + sitemap, TypeScript
- Шаблон блога с главной страницей, RSS, стилями
- Промпты для генерации статей (formal/casual стили)
- CLAUDE.md с правилами проекта
- Репозиторий: github.com/trinitispb-debug/ai-digest

## Этап 1 — Контент-фундамент

**Цель:** Первые статьи вручную, отладка формата.

- [ ] Написать 3-5 статей вручную в `src/content/blog/` по формату frontmatter
- [ ] Настроить страницу списка блогов (`/blog`)
- [ ] Настроить страницу отдельной статьи (`/blog/[slug]`)
- [ ] Проверить RSS (`/rss.xml`)
- [ ] Подобрать теги: `кии`, `лазерное-сканирование`, `реверс-инжиниринг`, `поверка`, `контрольно-измерительные-приборы`

## Этап 2 — Автоматический поиск новостей

**Цель:** Claude Code сам находит релевантные новости.

- [ ] Подключить Tavily MCP для поиска
- [ ] Настроить поисковые запросы по метрологическим темам
- [ ] Создать скрипт `scripts/find-news.ts` — ищет новости через API, фильтрует по релевантности
- [ ] Результат: JSON-файл со списком найденных статей и ссылками

## Этап 3 — Генерация контента

**Цель:** Из ссылки на новость → готовая статья в markdown.

- [ ] Скрипт `scripts/generate-post.ts` — принимает URL, генерирует статью через промпт
- [ ] Выбор стиля через аргумент (`--style formal` / `--style casual`)
- [ ] Автоматический frontmatter: title, description, pubDate, tags, source
- [ ] Проверка уникальности (не дублировать уже существующие статьи)
- [ ] Генерация обложек через Replicate или Pollinations

## Этап 4 — Сборка и деплой

**Цель:** Push в main → Vercel автоматически деплоит.

- [ ] Подключить Vercel к репозиторию
- [ ] Настроить `vercel.json` (если нужно)
- [ ] Проверить сборку `npm run build` локально
- [ ] Настроить домен (если есть)

## Этап 5 — Полная автоматизация

**Цель:** Один запуск — весь цикл от поиска до публикации.

- [ ] Скрипт `scripts/publish-digest.ts` — объединяет поиск + генерацию + commit + push
- [ ] Запуск по расписанию (GitHub Actions cron или локальный cron)
- [ ] Логирование результатов в `docs/digest-log.md`
- [ ] Уведомление в Telegram о выходе нового дайджеста

## Этап 6 — Улучшение качества

**Цель:** Рост аудитории и качества контента.

- [ ] Категории и теги как отдельные страницы (`/tags/[tag]`)

- [ ] Поиск по статьям (client-side)
- [ ] Архив по месяцам
- [ ] SEO-оптимизация (meta, Open Graph, structured data)

## Риски

1. **Качество источника** — Tavily может находить нерелевантные или низкокачественные статьи. Решение: фильтрация по доменам и ручная проверка на первых этапах.
2. **Точность терминологии** — LLM может использовать кальки с английского. Решение: строгие инструкции в промптах + CLAUDE.md правила.
3. **Объём контекста** — при большом количестве новостей может не хватать контекстного окна. Решение: обработка по одной статье за раз, compact между итерациями.
4. **Дублирование** — риск сгенерировать статью на тему, которая уже была. Решение: проверка по существующим файлам в `src/content/blog/`.
